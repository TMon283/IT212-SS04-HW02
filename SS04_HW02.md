# BÀI 2: Tối ưu Prompt (Thực hành viết Prompt giải nghĩa Stack Trace và gỡ lỗi)

## Nội dung Prompt sau khi tối ưu

```text
Bạn hãy đóng vai một Chuyên gia gỡ lỗi Java (Java Debugger) có nhiều năm kinh nghiệm.

Nhiệm vụ:
- Phân tích lỗi dựa trên mã nguồn và Stack Trace được cung cấp.
- Giải thích nguyên nhân gốc rễ (Root Cause) gây ra lỗi.
- Xác định chính xác dòng mã phát sinh lỗi.
- Đề xuất cách sửa chữa an toàn và đúng chuẩn OOP.

Bối cảnh:
Tôi đang phát triển một ứng dụng Java quản lý người dùng và chương trình bị sập với lỗi NullPointerException.

Ràng buộc:
- Phải giải thích dựa trên Stack Trace thực tế.
- Không được sửa bằng cách để users ở phạm vi public.
- Phải giữ nguyên tính bao đóng (Encapsulation).
- Phải khởi tạo danh sách bằng ArrayList.
- Trả về mã nguồn Java hoàn chỉnh sau khi sửa.

Mã nguồn:

import java.util.List;

public class UserManager {

    private List<String> users;

    public void addUser(String user) {

        users.add(user);

    }

}

Stack Trace:

Exception in thread "main" java.lang.NullPointerException:
Cannot invoke "java.util.List.add(Object)"
because "this.users" is null

    at UserManager.addUser(UserManager.java:7)
    at Main.main(Main.java:6)
```

## Mã nguồn Java hoàn chỉnh và an toàn do AI sinh ra

```java
import java.util.ArrayList;
import java.util.List;

public class UserManager {

    // Giữ tính bao đóng bằng cách để private
    private final List<String> users;

    // Khởi tạo danh sách ngay trong constructor
    public UserManager() {
        this.users = new ArrayList<>();
    }

    public void addUser(String user) {
        users.add(user);
    }

    public List<String> getUsers() {
        return new ArrayList<>(users);
    }
}
```

### Giải thích nguyên nhân lỗi

* Biến `users` được khai báo nhưng chưa được khởi tạo.
* Trong Java, biến tham chiếu chưa khởi tạo sẽ có giá trị mặc định là `null`.
* Khi gọi:

```java
users.add(user);
```

chương trình thực tế đang cố thực hiện:

```java
null.add(user);
```

nên phát sinh `NullPointerException`.

Dựa vào Stack Trace:

```text
at UserManager.addUser(UserManager.java:7)
```

có thể xác định dòng gây lỗi chính là lệnh `users.add(user);`.

Giải pháp an toàn nhất là khởi tạo danh sách trong constructor bằng `ArrayList`, đồng thời giữ thuộc tính ở mức `private` để đảm bảo nguyên tắc Encapsulation của lập trình hướng đối tượng.
